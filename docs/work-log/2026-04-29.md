### 배경
대시보드 견적서 세부사항 합계가 견적 금액과 맞지 않는 문제 제보 → 사이트 전체 계산식 전수 점검 진행

### 수정 내용

#### 1. [치명] 견적 상세 항목 합계 ↔ 견적 금액 불일치
- **파일**: `components/dashboard/ProjectForm.tsx`, `ProjectForm.module.css`
- **문제**: `amount`(견적 총액)와 `amountDetails`(항목별 내역)가 완전히 독립적. 상세 항목을 추가해도 총액에 반영 안 됨
- **수정**: 항목 합계 실시간 표시 + 불일치 시 경고 메시지 + "합계로 견적 반영" 버튼 추가

#### 2. [치명] Stats vs Insights 매출 데이터 소스 불일치
- **파일**: `components/dashboard/Stats.tsx`
- **문제**: Stats는 projects 테이블(settlementStatus 기준), Insights는 settlements 테이블(category 기준)로 매출 계산 → 같은 대시보드에서 매출이 두 가지 다른 숫자로 표시
- **수정**: Stats를 settlements 테이블 기준으로 변경하여 Insights와 동일 기준 적용

#### 3. [치명] work/page.tsx 매출 계산 기준 불일치
- **파일**: `app/dashboard/work/page.tsx`
- **문제**: settlements 전체(확정+예정)를 합산해서 totalRevenue로 사용 → Stats/Insights와 또 다른 수치
- **수정**: 확정 정산(`category === "확정"`)만 사용하도록 통일

#### 4. [중간] Insights pending 필터 누락
- **파일**: `components/dashboard/Insights.tsx`
- **문제**: `category === "예정"`만 pending으로 필터 → "확정"도 "예정"도 아닌 카테고리가 있으면 매출에서 누락
- **수정**: `category !== "확정"` 전체를 pending으로 변경 (Stats와 동일)

#### 5. [중간] Insights 월수 계산 오류
- **파일**: `components/dashboard/Insights.tsx`
- **문제**: 비용이 없으면 `months = 2` 하드코딩, 비용 첫 날짜만 참조
- **수정**: 비용+정산 모든 날짜에서 가장 이른 날짜 ~ 현재까지 포함 월수로 계산, 기본값 1

#### 6. [낮음] 비용 비율 경고 0 나누기
- **파일**: `components/dashboard/Insights.tsx`
- **문제**: `confirmedRevenue`가 0이면 `(totalExpense / confirmedRevenue) * 100` → Infinity/NaN
- **수정**: `confirmedRevenue > 0` 가드 추가
